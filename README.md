# Main Site 主站

这是 `nabr.cn` 的主站前端仓库。

它本身是一个 **静态站点**，负责首页展示、导航、登录/注册弹窗、主题切换和简单管理页；实际的用户认证、访客记录等接口由后端服务提供，通常通过 **Nginx 反向代理** 与个人空间项目配合使用。

## 项目定位

这个仓库主要负责：

- 主站首页展示
- 导航到个人空间、博客、杂谈、AI 页面
- 登录 / 注册弹窗交互
- 登录态显示与退出
- 管理页入口与超管访客查看页面
- 亮暗主题与移动端导航

它不是一个独立后端项目，单独部署时只能提供静态页面；要完整使用登录、访客、管理等能力，需要后端 API 配合。

## 功能特性

- **首页展示**：简洁 landing page，带主入口按钮
- **统一导航**：跳转到空间、公告、博客、杂谈、AI 助手
- **登录 / 注册弹窗**：前端直接调用 API 完成认证
- **登录态同步**：支持主站与空间共用登录状态
- **管理页**：超管可查看最近访客
- **亮暗主题**：支持主题初始化与切换
- **移动端适配**：汉堡菜单、移动端导航层
- **无构建依赖**：纯 HTML / CSS / JavaScript，可直接托管

## 技术栈

- **页面**：HTML
- **样式**：CSS
- **交互**：原生 JavaScript
- **部署方式**：Nginx / 任意静态文件服务

## 页面说明

### `/`
主站首页，提供：

- 品牌展示
- 进入空间按钮
- AI 助手入口
- GitHub 链接
- 登录 / 注册弹窗
- 当前用户信息显示

### `/admin.html`
主站管理页。

前端会先请求 `/api/me` 判断当前用户是否为 `superadmin`，通过后再请求 `/api/visitors` 拉取访客记录。

## 依赖的后端接口

主站前端默认调用这些接口：

- `POST /api/login`
- `POST /api/register`
- `GET /api/captcha`
- `GET /api/me`
- `POST /api/logout`
- `GET /api/visitors`

所以部署时需要保证这些 `/api/*` 请求能被正确转发到后端服务。

## 推荐部署方式

最常见的做法是：

- 主站仓库静态文件放在 Nginx 可访问目录
- `/` 直接提供 `main-site/public` 下内容
- `/api/` 反向代理到个人空间后端
- `/space/` 反向代理到个人空间应用
- `/blog`、`/chitchat`、`/ai/` 等路径按你的整体站点结构继续转发

也就是说，这个仓库通常是 **整站入口层**，而不是完整业务后端。

## 本地预览

因为它是纯静态前端，你可以直接用任意静态服务器预览，例如：

```bash
cd public
python3 -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

注意：

- 如果没有配后端接口，登录 / 注册 / 管理页相关功能不会正常工作
- 只看界面和样式的话，静态预览就够了

## 与个人空间项目的关系

这个仓库与 `personal-space` 是配套使用的：

- `main-site`：主站入口、首页、登录弹窗、管理页
- `personal-space`：动态、文章、用户、通知、公告、API 等核心业务

如果你只想改首页观感、导航、登录弹窗样式，主要改这个仓库；
如果你要改动态、评论、用户系统、公告接口，主要改 `personal-space`。
